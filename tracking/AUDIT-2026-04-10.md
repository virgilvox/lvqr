# Audit Notes - 2026-04-10

## What's genuinely proven (with real tests)

- MoQ relay accepts QUIC connections, fans out tracks to 3 subscribers (3 integration tests)
- RTMP ingest: full pipeline from RTMP client -> bridge -> MoQ subscriber (2 integration tests)
- Mesh coordinator builds balanced trees of 50 nodes, handles orphan reassignment (13 tests)
- Admin HTTP endpoints return real stats from relay metrics and bridge state (4 tests)
- Core data structures (ring buffer, GOP cache, registry) work correctly (25 tests)
- Python admin client API is correct (8 tests)
- Registry fanout benchmarked: ~230ns to publish to 500 subscribers (criterion)

## Critical gaps found and fixes applied

### 1. Admin API was reading from empty Registry -- FIXED
- Admin now uses callback-based `AdminState` wired to real relay `RelayMetrics` and `RtmpMoqBridge` state
- Reports actual connection counts, active RTMP publishers, and stream names
- Tests updated to verify against real-style state

### 2. No RTMP end-to-end test -- FIXED
- Added `rtmp_bridge_integration` test with 2 test cases
- `rtmp_publish_creates_moq_broadcast`: full RTMP handshake + connect + publish + video data, verifies MoQ subscriber reads the keyframe
- `rtmp_audio_data_reaches_moq`: same flow for audio data
- Tests use rml_rtmp ClientSession as a real RTMP client over TCP

### 3. lvqr-core is disconnected from the relay -- DOCUMENTED
- Added module-level docs to lvqr-core explaining its actual role
- Registry/RingBuffer/GopCache are standalone data structures, not in the relay hot path
- Relay uses moq-lite's OriginProducer directly
- These exist for shared types and future use (WS-fMP4 fallback, stats aggregation)

### 4. Browser client can't play video -- DOCUMENTED IN README
- `@lvqr/core` and `@lvqr/player` connect but MoQ subscribe framing is not implemented
- lvqr-wasm doesn't compile (web-sys WebTransport types missing)
- README now clearly states this in the Status section

### 5. README overpromised -- FIXED
- Added "Status (v0.1.0)" section listing what works and what doesn't
- Removed misleading "5000+ viewers" claim as a proven metric (now marked as design target)
- Removed browser playback from quickstart
- Client library table now shows per-library status

### 6. Zero benchmarks -- FIXED
- Added `fanout` benchmark: registry publish to N subscribers, varying payload sizes, multi-track
- Results: ~230ns per publish to 500 subscribers (4KB frames), ~27us for 50 tracks x 10 subs

## Remaining gaps

### Browser playback (major feature gap)
- Needs MoQ subscribe protocol implemented in WASM/TypeScript
- lvqr-wasm needs web-sys WebTransport feature flags fixed
- Consider using moq-dev's `@moq/lite` TypeScript package instead of custom implementation

### Mesh media relay (major feature gap)
- Coordinator assigns tree positions but no code relays media between peers
- Signal server forwards messages but nothing generates SDP offers
- `_mesh` variable in CLI is unused

### Per-stream subscriber counts
- Admin API currently returns `subscribers: 0` for individual streams
- Needs moq-lite to expose per-broadcast subscriber state, or manual tracking in the bridge

### io_uring zero-copy path
- Feature flag exists but no implementation

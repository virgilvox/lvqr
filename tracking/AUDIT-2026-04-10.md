# Audit Notes - 2026-04-10

## What's genuinely proven (with real tests)

- MoQ relay accepts QUIC connections, fans out tracks to 3 subscribers (3 integration tests)
- Mesh coordinator builds balanced trees of 50 nodes, handles orphan reassignment (13 tests)
- Admin HTTP endpoints return correct responses (4 tests)
- Core data structures (ring buffer, GOP cache, registry) work correctly (25 tests)
- Python admin client API is correct (8 tests)

## Critical gaps found

### 1. lvqr-core is disconnected from the relay
- Registry, RingBuffer, GopCache are never used in the actual data path
- Relay uses moq-lite OriginProducer which has its own internal routing
- Admin API reads from empty Registry, always shows 0 tracks/0 subscribers
- Fix: Wire admin stats to relay metrics or Origin state

### 2. No RTMP end-to-end test
- RtmpMoqBridge has 0 tests
- The OBS -> RTMP -> relay -> MoQ subscriber path is unproven
- Fix: Integration test that sends RTMP handshake + FLV data, verifies MoQ subscriber receives it

### 3. Mesh is tree-assignment algorithm only
- No code relays media between browser peers
- Signal server forwards messages but nothing generates SDP offers server-side
- _mesh variable in CLI is unused (underscore prefix)
- Fix: This is a major feature gap, not a bug

### 4. Browser client can't play video
- LvqrClient.subscribe() is a no-op comment
- No MoQ protocol framing in WASM or TypeScript
- <lvqr-player> connects but renders nothing
- Fix: Implement MoQ subscribe protocol or use moq-dev's @moq/lite TS package

### 5. README overpromises
- Fix: Add "Status" section clearly marking what works vs. planned

### 6. Zero benchmarks
- Fix: Add a simple throughput benchmark (1 pub, N subs)
